# RainSweep 設計ノート

Raindrop.io のリンク切れ（ハンガーマーク相当）を自動掃除するツールの設計。

## 調査結果
- APIでの削除 (`DELETE /raindrop/{id}`) は、デフォルトで **ゴミ箱 (Trash, ID: -99)** へ移動する。
- 永久削除はゴミ箱内で削除を実行した場合のみ発生。安全性が高い。
- 無料プランではAPIから `broken` ステータスを直接引けない可能性があるため、独自にリンクチェックを行う。

## 実装方針
- **言語**: Python 3.12+ (uv管理)
- **判定ロジック**: 「C: リトライあり」
  - HTTP HEAD/GET リクエストを送信。
  - 失敗した場合、一定時間（例: 5秒）待機して再試行。
  - 2回（または指定回数）失敗した場合のみ「リンク切れ」と判定。
- **削除挙動**: 「B: ゴミ箱へ移動」
  - APIの `DELETE` を使用し、安全にゴミ箱へ送る。
- **実行形態**: 「A: 手動実行」
  - CLIツールとして実装。

## 依存ライブラリ
- `raindrop-io-py`: API操作
- `httpx`: 非同期HTTP通信
- `python-dotenv`: トークン管理

## セキュリティ
- `RAINDROP_TOKEN` は `.env` に格納。
- `.env` は `.gitignore` で確実に除外。
